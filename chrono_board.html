📖 ChronoBoard 사용설명서

코르크보드 위에 일정·메모·작업공간을 붙이고, 구글 시트로 어디서나 함께 보세요.
아래에서 ⭐ 표시된 구글 시트 연동 부분을 특히 자세히 정리했습니다.
🧷 기본 사용법
➕ 추가상단 ‘추가’에서 일정·메모·작업공간·파일 카드를 보드에 붙입니다.
🖱️ 이동·크기카드를 끌어 옮기고, 오른쪽 아래 모서리를 잡아 크기를 바꿉니다.
⭐ 중요카드의 ☆를 누르면 오른쪽 ‘중요’ 액자에 보관됩니다.
⧉ 복사 / ✕ 삭제카드 위 단추로 내용을 복사하거나 카드를 지웁니다.
↕ 정렬‘만든 순서 / 종류별’로 카드를 깔끔히 정돈합니다.
🔍 검색오른쪽 위 검색창으로 카드 내용을 찾습니다.
🗂️ 작업공간 · 🏥 병원 일정판 · 🔔 알림
작업공간카드의 ⤢로 크게 열어 글과 항목을 정리하고, 여러 ‘페이지’로 이어 씁니다.
병원 일정판상담·수술 등 여러 명을 ‘＋ 추가’로 적고, ↺ 초기화로 다음 날 새로 시작합니다. ▾로 최소화됩니다.
알림매일·매주·매월·특정일로 알림을 등록하면 시각에 포스트잇과 알림이 뜹니다.
💾 데이터는 어디에 저장되나요?

모든 카드는 이 브라우저 안에 자동 저장됩니다(같은 기기·같은 브라우저에서 유지). 상단의 용량 표시로 사용량을 볼 수 있습니다. 다른 기기(집↔병원)에서 함께 보려면 아래 구글 시트 연동을 쓰세요.

⭐ 구글 시트 연동 — 자동저장 & 가져오기
자동
📊 시트 자동저장
연동해두면 일정·메모를 만들 때마다 구글 시트에 자동으로 한 줄씩 쌓입니다. 작업공간은 크게 연 화면의 📊 시트로 전송 버튼을 누를 때 보냅니다. 시트에는 시각·종류·제목·내용 순으로 기록됩니다.
가져옴
📥 시트 불러오기
상단 📥 시트 불러오기를 누르면 시트에 쌓인 내용을 보드로 가져옵니다. ‘추가로 불러오기’(지금 카드 유지)와 ‘새로 불러오기’(보드 비우고 새로) 중 선택합니다. 집에서도 같은 시트 주소만 넣으면 병원 메모를 그대로 봅니다.
💡 사진·파일 첨부는 시트로 오가지 않고, 글(텍스트)만 동기화됩니다.
⭐ 세팅 방법 (처음 한 번만)

코딩은 필요 없습니다. 아래 순서대로 따라 하시면 됩니다. 컴퓨터(PC)에서 하시는 걸 권합니다.

  1. 1
    빈 구글 시트 만들기 — sheets.google.com → ＋(빈 스프레드시트). 머리글은 안 적어도 됩니다(자동 생성).
  2. 2
    스크립트 편집기 열기 — 시트 메뉴 확장 프로그램 → Apps Script.
  3. 3
    코드 붙여넣기 — 기존 내용을 모두 지우고, 아래 ‘쓰기(doPost)’와 ‘읽기(doGet)’ 코드를 함께 붙여넣습니다. 시트ID 부분만 본인 시트 주소(/d/와 /edit 사이 글자)로 바꾸세요. → 💾 저장.
  4. 4
    웹앱으로 배포 — 오른쪽 위 배포 → 새 배포 → 유형: 웹 앱. 액세스 권한 = ‘모든 사용자(Anyone)’ 로 꼭 바꾸고 배포. ★ ‘모든 사용자’가 아니면 저장이 안 됩니다.
  5. 5
    권한 허용(처음 한 번) — 계정 선택 → ‘고급’ → ‘…(안전하지 않음)으로 이동’ → 허용.
  6. 6
    주소 복사 — …/exec 로 끝나는 웹앱 URL을 복사합니다.
  7. 7
    ChronoBoard에 붙여넣기 — 작업공간을 크게 열고 ⚙ → 주소 붙여넣기 → ✉️ 테스트 전송 → 시트에 ‘테스트’ 줄 확인 → 저장.
📋 붙여넣을 코드 (쓰기 + 읽기) 
function doPost(e) {
  var ss = SpreadsheetApp.openById('시트ID');
  var sheet = ss.getSheets()[0];
  if (sheet.getLastRow() === 0)
    sheet.appendRow(['시각','종류','제목','내용']);
  var d = {};
  if (e && e.postData && e.postData.contents)
    d = JSON.parse(e.postData.contents);
  sheet.appendRow([new Date(), d.kind||'', d.title||'', d.content||'']);
  return ContentService.createTextOutput(
    JSON.stringify({result:'ok'})).setMimeType(ContentService.MimeType.JSON);
}

function doGet(e) {
  var ss = SpreadsheetApp.openById('시트ID');
  var v = ss.getSheets()[0].getDataRange().getValues();
  var rows = [];
  for (var i = 1; i < v.length; i++)
    rows.push({ time: v[i][0] ? new Date(v[i][0]).toLocaleString() : '',
                kind: v[i][1]||'', title: v[i][2]||'', content: v[i][3]||'' });
  return ContentService.createTextOutput(
    JSON.stringify(rows)).setMimeType(ContentService.MimeType.JSON);
}
🔁 코드를 고친 뒤에는 배포 → 배포 관리 → 연필(수정) → 버전 ‘새 버전’ → 배포를 해야 반영됩니다. (주소는 그대로 유지)
🛠️ 잘 안 될 때
저장(테스트)이 안 돼요배포 권한이 ‘모든 사용자’인지, 주소가 /exec로 끝나는지 확인. 실행기록(시계 아이콘)에서 doPost 오류를 보세요.
불러오기가 안 돼요doGet을 추가했는지, ‘새 버전’으로 다시 배포했는지 확인. 주소를 브라우저에 넣어 [로 시작하는 데이터가 보이면 정상입니다.
오류율 100%스크립트의 시트ID가 본인 시트로 정확히 바뀌었는지 확인하세요.
