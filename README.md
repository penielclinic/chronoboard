# chronoboard
# 📌 ChronoBoard

> Think, plan, and track — all in one place.

코르크보드 감성의 **단일 HTML 파일** 메모/일정/작업공간 보드입니다. 포스트잇을 핀으로 꽂듯 카드를 붙이고, 드래그로 옮기고, 검색·정렬·복사하고, 작업공간을 전체화면으로 펼쳐 그 안에 다시 카드를 붙일 수 있습니다. 모든 데이터는 브라우저의 **localStorage**에 자동 저장됩니다.

설치도, 빌드도, 서버도 필요 없습니다. `chrono_board.html` 하나면 끝입니다.

---

## ✨ 주요 기능

- **4가지 카드 타입** — 일정(노랑) · 메모(블루) · 작업공간(그린) · 파일(보라). 색으로 한눈에 구분됩니다.
- **핀으로 고정되는 포스트잇** — 추가하면 코르크보드에 핀으로 꽂힌 카드로 나타납니다.
- **드래그 이동 & 크기 조절** — 모든 카드를 자유롭게 옮기고, 우측 하단 핸들로 크기를 조절합니다.
- **파일 첨부** — 그림·문서(PDF, Word, Excel, PPT, 한글 등)를 업로드. 이미지는 썸네일 + 라이트박스 미리보기, 모든 파일은 **열기 / 저장** 지원.
- **전체화면 작업공간** — 작업공간 카드를 화면의 90% 크기로 확대. 왼쪽엔 자유 텍스트 영역, 오른쪽 캔버스엔 일정·메모·파일을 다시 붙일 수 있습니다.
- **검색** — 단어를 입력하면 제목·내용·날짜·파일명에 일치하는 포스트잇만 보드에 남습니다.
- **정렬** — 만든 순서 / 종류별로 카드를 격자 정돈.
- **전체 복사** — 카드의 모든 내용을 클립보드로 한 번에 복사. 작업공간은 내부 항목까지 함께 복사됩니다.
- **아날로그 시계 위젯** — 실시간 시·분·초침 + 오늘 날짜.
- **자동 저장** — 변경할 때마다 localStorage에 저장되어 새로고침해도 그대로 유지됩니다.

## 🚀 사용법

### 가장 빠른 방법
1. `chrono_board.html`을 내려받습니다.
2. 브라우저로 파일을 엽니다 (더블클릭).
3. 끝. 인터넷 연결도 필요 없습니다.

### GitHub Pages로 배포하기
1. 이 저장소를 GitHub에 올립니다.
2. **Settings → Pages → Branch** 를 `main` / `root` 로 지정합니다.
3. `https://<사용자명>.github.io/<저장소명>/chrono_board.html` 로 접속합니다.

> 파일 이름을 `index.html`로 바꾸면 저장소 주소만으로 바로 열립니다.

## 💾 데이터 저장 (localStorage) — 자세히

ChronoBoard는 **별도의 서버나 계정 없이** 브라우저에 내장된 `localStorage`에 데이터를 저장합니다. 아래 내용을 이해하면 데이터가 어떻게 보관되고, 어떤 한계가 있는지 명확히 알 수 있습니다.

### 1) 무엇이, 언제 저장되나

저장 키는 `chronoboard.v1` 하나이며, 다음 동작이 일어날 때마다 자동 저장됩니다.

- 카드 추가 / 삭제
- 카드 드래그 이동, 크기 조절
- 정렬 실행
- 작업공간의 제목·본문 입력 (입력 중에는 0.4초 디바운스로 묶어 저장)
- 작업공간 안 카드의 추가 / 삭제 / 이동
- 페이지를 닫기 직전(`beforeunload`)

저장되는 내용은 각 카드의 **종류, 제목, 내용, 날짜/메타, 첨부 파일, 좌표(x·y), 크기(w·h), 생성 순서**이며, 작업공간 카드는 내부 텍스트와 붙인 항목까지 함께 저장됩니다.

### 2) 저장 구조 (JSON)

```json
{
  "savedAt": 1719100000000,
  "cards": [
    {
      "type": "schedule",
      "title": "마케팅팀 미팅",
      "meta": "6월 25일 13:00 - 13:45",
      "body": "회의실 A",
      "file": null,
      "x": 120, "y": 80, "w": 240, "h": 150,
      "order": 0
    },
    {
      "type": "work",
      "title": "이번 주 캠페인",
      "x": 400, "y": 60, "w": 340, "h": 230,
      "order": 1,
      "ws": {
        "title": "이번 주 캠페인",
        "text": "자유롭게 적는 메모…",
        "items": [
          { "type": "memo", "title": "아이디어", "body": "...", "x": 60, "y": 50 }
        ]
      }
    }
  ]
}
```

### 3) 첨부 파일은 어떻게 저장되나

업로드한 파일은 `FileReader`로 읽어 **Base64 data URL** 형태로 JSON 안에 그대로 담깁니다. 덕분에 서버 없이도 이미지·문서가 보드에 영구적으로 남습니다. 다만 Base64는 원본보다 약 **33% 더 큰** 용량을 차지합니다.

### 4) ⚠️ 용량 한계와 자동 대응

`localStorage`는 도메인(또는 파일)당 보통 **약 5MB**로 제한됩니다. 텍스트 카드는 수백~수천 개를 넣어도 문제없지만, **이미지·문서를 여러 개 첨부하면 한도를 넘길 수 있습니다.**

ChronoBoard는 저장이 실패(quota 초과)하면 다음 순서로 **자동 대응**합니다.

1. 먼저 모든 데이터를 그대로 저장 시도.
2. 실패하면 → **무거운 파일 데이터(Base64)만 제외**하고 나머지(텍스트·위치·구조)는 저장. 이때 툴바에 `⚠ 일부 파일은 용량 초과로 저장 안 됨` 표시.
3. 그래도 실패하면 → `⚠ 저장 실패 (저장공간 가득 참)` 표시.

> 파일이 제외되어 저장된 경우, 새로고침하면 해당 **파일 카드는 남지만 첨부 내용은 비어 있습니다.** 큰 파일은 보드에 두기보다 원본을 따로 보관하시길 권합니다.

### 5) 저장 위치와 범위

- 데이터는 **이 브라우저, 이 기기에만** 저장됩니다. 다른 브라우저·다른 PC·시크릿 모드와는 공유되지 않습니다.
- 같은 파일이라도 **여는 경로(file:// 경로나 배포 URL)가 다르면** 저장 공간이 분리됩니다.
- 브라우저의 \"인터넷 사용 기록 삭제\" 시 함께 지워질 수 있습니다.

### 6) 데이터 초기화

툴바의 **전체 비우기** 버튼을 누르면 보드의 모든 카드와 저장된 데이터(`chronoboard.v1`)가 삭제됩니다. (확인 창이 한 번 뜹니다. 되돌릴 수 없습니다.)

### 7) 백업 / 옮기기 (수동)

현재 버전은 파일 내보내기 버튼이 없습니다. 데이터를 백업하거나 다른 기기로 옮기려면 브라우저 개발자 도구(F12) 콘솔에서:

```js
// 백업: 출력된 내용을 복사해 텍스트 파일로 저장
copy(localStorage.getItem('chronoboard.v1'));

// 복원: 백업해둔 JSON 문자열을 붙여넣기
localStorage.setItem('chronoboard.v1', '<백업한 JSON>');
location.reload();
```
